## Key Learnings

- **Project:** errata-stickers-generator
- **Coordinate model:** Region.bbox is stored in PDF user-space points (origin bottom-left) so it feeds pdf-lib embedPage/drawPage directly. pdfjs `viewport.convertToPdfPoint(x,y)` maps canvas→PDF and is invariant to on-screen render scale. RegionLayer works in 0..1 fractions of the page so overlays survive CSS scaling.
- **Bleed semantics:** footprint = bboxW*scale + 2*bleedPt. In generate.ts, expand the source clip bbox by `bleedPt/scale` (so after drawing at `scale` the bleed equals bleedPt). No clamping to page bounds in MVP — bleed beyond the page is transparent.
- **Rotation placement (pdf-lib):** drawPage/drawImage rotate=90° maps local (u,v)→(x-v, y+u). For a rotated footprint, draw at x = leftEdge + footprintW, y = bottomEdge. Verified headlessly. Same convention for drawImage.
- **Output generation rasterises regions via pdf.js, NOT pdf-lib embedPage** (changed 2026-06-19). embedPage silently dropped/blackened images for a real errata PDF whose icons are DCTDecode images with SMask soft-masks nested inside Form XObjects with transparency `/Group` (ICCBased cs). generate.ts now renders each region through pdf.js at RENDER_DPI=300 of the printed size (region rendered directly via a translation transform — no full-page canvas), `canvas.toBlob`→PNG→`embedPng`, drawn at the placement. WYSIWYG with the editor (same engine). Trade-off: raster not vector; rasterised once per region id, reused for copies. generatePdf is browser-only now (document.createElement) — can't headless-test the full pipe, but rasterRegion logic verified with @napi-rs/canvas (transitive dep) against the real PDF.
- **Packing:** maxrects-packer; usable bin = sheet − 2*margin, gap = packer padding, placements offset by margin afterward. bin.rects swap width/height when rect.rot is true.
- **pdfjs in Vite:** worker via `import workerUrl from 'pdfjs-dist/build/pdf.worker.min.mjs?url'` + `GlobalWorkerOptions.workerSrc = workerUrl`. pdfjs detaches the buffer it parses, so pass `bytes.slice()` and keep an original copy for pdf-lib.
- **OpenWolf anatomy.md is auto-regenerated** (a scan hook rewrites it with per-file token estimates) — no need to hand-maintain it; it may transiently list deleted /tmp files.
- **Region editing (RegionLayer):** interaction runs in device-px (viewport.scale = px per pt; viewport.width = pageWidthPt*scale). State machine create|move|resize. Pointer capture is set on the overlay element from child (region/handle) pointerdown handlers, with stopPropagation so the background create doesn't also fire. Commit px rect → bbox via viewportRectToPdfBBox only on pointerup (move skipped if delta <1px = pure select).
- **8 resize handles** positioned by cx/cy in {0,.5,1} with translate(-50%,-50%); resizeRect() clamps each dragged edge to keep min size (mmToPt(2)) and stay within [0,maxW/H]. Tailwind cursor-nwse-resize/nesw-resize/ns/ew utilities are detected from literal class strings.
- **RegionInspector** is a floating panel rendered in PageViewer as a sibling AFTER RegionLayer (higher stacking → owns its pointer events; also stopPropagation as belt-and-suspenders). Y field is "from top" = ptToMm(pageHeightPt - bbox.top); pageHeightPt from viewportPageSizePt(viewport.viewBox). Editing X/Y anchors top-left; W grows right, H grows down.
- **Keyboard editing** lives in a window keydown listener inside RegionLayer; it no-ops when focus is in INPUT/TEXTAREA so inspector typing (Delete/Backspace) doesn't nuke the region.
- **Crop marks** are a SheetSettings toggle (`cropMarks`, default false). drawCropMarks() in generate.ts draws 8 short L-strokes at the trim-box corners (footprint inset by bleed), offset outward by the bleed, via pdf-lib `page.drawLine`. Trim box = footprint − bleed on each side.
- **Navigation is a tree** (`RegionTree.tsx`): Document → Page → Region, replacing the old PageNavigator grid + flat RegionList (both deleted). Expand state in a local Set of keys (`docId`, `docId:pageIndex`); active page auto-expanded via effect. Region rows have ⧉ duplicate (modal `DuplicateDialog` picks target doc+page; same-page copy offset by mmToPt(4)) and ✕ delete (inline ✓/✗ confirm, reused for doc removal too).
- **Project save/load is JSON-only, PDFs re-attached by filename** (`lib/project.ts` + `ProjectControls` in Toolbar). Regions reference docs by `name` (runtime docId is unstable). Open: pick .json → resetProject() → setSheet → if docs needed, prompt PDF multi-picker → match by file.name → loadPdfFile/addDocs → remap docName→docId → addRegion each; unmatched names warned. Two hidden file inputs chained via a pendingRef.
- **Generation = variants, not one PDF.** `generateVariants(regions, sheet, 10)` sweeps ordering strategies × both PACKING_LOGIC values + 12 seeded shuffles (mulberry32 in `lib/random.ts`), dedupes by **cost** (sheets+round(eff*1000)+unplaced — NOT arrangement, since cosmetic position diffs are meaningless for a text list), sorts fewest-sheets then highest efficiency. ResultPreview lists them; per-row Download calls generatePdf on that variant's stored PackResult. packRegions() is now a thin default over packRegionsWith().
- **Region.quantity** = number of identical copies in the layout (≥1, default 1). packRegionsWith pushes `quantity` items with the same regionId; generatePdf is unchanged (embed cached per regionId, drawn per placement). Editable via "Copies" in RegionInspector; shown as `×N` in RegionTree; persisted in project (read with `?? 1`). Distinct from ⧉ duplicate (which makes a separate editable region).
- **Per-region margin (`Region.marginMm?`, default 0 via `?? 0`)** = an EMPTY layout band, NOT extra bleed. `stickerFootprintPt` adds `mmToPt(marginMm)` to the bleed pad so the packer reserves more space (neighbour ends up `gap + margin` away). The captured content (rasterRegion/embedFor) is unchanged; generate.ts insets the DRAW by `marginPt` (innerLeft/innerBottom, imgW=p.w−2m, innerFootW for the rotation anchor) so the sticker is centered and the band stays blank. Crop marks use the inner footprint. Edited via "Margin, mm" in RegionInspector (3-col grid with Scale/Copies).
- **Project save fingerprints used PDFs:** `buildProject` is now ASYNC — serialises ONLY docs that carry a region (`usedIds`), each with `hash: await sha256Hex(d.bytes)` (`lib/hash.ts`, WebCrypto SHA-256; must copy via `new Uint8Array(bytes)` to satisfy the BufferSource type under new TS lib). `ProjectFile.docs` entry = `{name,numPages,hash?}`; `ProjectRegion` carries `marginMm?`. Still `version:1` (additive optional fields → old files load fine). `handleSave` must `await`.
- **LoadProjectDialog supports drag-and-drop** (same pattern as `Uploader.tsx`: onDragOver/onDragLeave/onDrop + `dragging` highlight `border-sky-400 bg-sky-400/10`). Each row is a drop target (drop one PDF → attaches to THAT row, `stopPropagation` so the dialog-wide handler doesn't double-fire); dropping several files anywhere on the modal box auto-assigns each by saved name, else into the next empty row. Shared `applyAssignments({index,file}[])` hashes + places in one `setRows`. Overlay has `onDragOver/onDrop preventDefault` so a stray drop on the backdrop doesn't navigate the browser to the file. `isPdf` filter = `type==='application/pdf' || name endsWith .pdf`.
- **Load is now a dialog, not an auto file-picker** (`LoadProjectDialog.tsx`, replaces the old chained `pdfInputRef`/`onPdfsSelected` in ProjectControls). Lists used PDFs; user attaches EACH file individually (SLOT-based match keyed by the project doc name — renamed files still load); per-row SHA-256 check → ok/mismatch(amber)/unknown(no hash). `resetProject()`+`setSheet` are DEFERRED into `applyLoad()` so Cancel preserves current work. Load button: all attached → apply; some missing → inline confirm ("N not attached, skip those regions?") → Continue. Regions whose `docName` wasn't attached are skipped (non-blocking `setError` summary).
- **Store uses a `revision` counter** (bumped by addDocs/removeDoc/add/update/removeRegion/setSheet/resetProject) instead of the old `generated` field; ResultPreview clears its computed variants when `revision` changes. `resetProject()` disposes pdfjs docs.

- **GitHub Pages deploy** (project page) via GitHub Actions (`.github/workflows/deploy.yml` — configure/upload-pages-artifact/deploy-pages, no gh-pages branch; `npm ci` + Node 20). `vite.config.ts` sets `base` AUTO-DERIVED: `process.env.GITHUB_REPOSITORY?.split('/')[1]` → `/<repo>/` in CI, `/` locally. Needs `declare const process: { env: Record<string,string|undefined> }` in vite.config.ts because `@types/node` isn't installed (tsconfig.node would otherwise error TS2580). **Gotcha:** the pdf.js worker is a Vite `?url` asset (`lib/pdf/load.ts`), so it follows `base` automatically — wrong base = worker 404 = marking/generation silently no-op. `crypto.subtle` (hash feature) needs a secure context, which Pages HTTPS provides. Verify subpath locally: `GITHUB_REPOSITORY=u/repo npm run build && npx vite preview --base /repo/`.

## Decision Log
